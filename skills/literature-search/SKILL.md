# 文献检索与收集 (Literature Search)

## 技能概述

本技能专注于学术研究中的文献检索环节，帮助用户高效地从各类学术数据库中查找、筛选和管理相关文献，为后续的阅读分析和研究空白发现提供高质量的文献基础。

## 核心功能

### 1. 搜索策略制定

**研究问题转化**
- 将用户的研究兴趣或问题转化为可检索的研究问题（PICO/SPIDER框架）
- PICO框架（医学/社会科学）：Population（研究对象）、Intervention（干预措施）、Comparison（对照）、Outcome（结果）
- SPIDER框架（质性研究）：Sample（样本）、Phenomenon of Interest（感兴趣现象）、Design（研究设计）、Evaluation（评估）、Study type（研究类型）
- 确定关键概念和变量

**关键词提取与扩展**
- 从研究问题中提取核心关键词
- 生成同义词、近义词、相关术语
- 考虑不同拼写形式（美式/英式、单数/复数）
- 识别专业术语和缩写
- 使用MeSH主题词（医学）、IEEE Thesaurus（工程）等受控词汇表

**检索式构建**
- 布尔逻辑运算符（AND、OR、NOT）的正确使用
- 短语检索（引号包裹）
- 截词符和通配符（*、?）
- 字段限定（标题、摘要、关键词）
- 邻近算符（NEAR/x、ADJ/x）

### 2. 数据库选择与检索

**主要学术数据库**

| 数据库 | 适用领域 | 特点 |
|--------|---------|------|
| PubMed/MEDLINE | 生物医学 | 免费、覆盖广、MeSH主题词 |
| Web of Science | 多学科 | 引文索引、影响因子、JCR分区 |
| Scopus | 多学科 | 覆盖最广、作者标识符、CiteScore |
| IEEE Xplore | 工程/计算机 | 会议论文、技术标准 |
| ACM Digital Library | 计算机科学 | 会议、期刊、杂志 |
| Google Scholar | 多学科 | 覆盖面广、灰色文献、引用追踪 |
| CNKI | 中文文献 | 中国知网、中文期刊学位论文 |
| Web of Science Core Collection | 多学科 | SCI/SSCI/AHCI索引 |
| Embase | 生物医学/药学 | 药物索引、欧洲文献 |
| Cochrane Library | 循证医学 | 系统综述、临床试验 |
| PsycINFO | 心理学 | APA主题词、心理测量 |
| ERIC | 教育学 | 教育理论与实践 |
| JSTOR | 人文社科 | 过刊档案、延后开放 |

**检索策略执行**
- 为每个数据库定制检索式
- 考虑各数据库的语法差异
- 执行检索并记录检索结果数量
- 保存检索策略以便复现

### 3. 文献筛选与管理

**纳入与排除标准**
- 时间范围（如近5年、近10年）
- 研究设计类型（随机对照试验、队列研究、案例研究等）
- 语言限制
- 研究对象特征（年龄、疾病、地域等）
- 文献类型（同行评审期刊、会议论文、预印本、书籍章节等）
- 质量门槛（最低影响因子、引用次数等）

**筛选流程**
- 标题和摘要初筛（快速排除明显不相关文献）
- 全文复筛（深入评估纳入标准）
- 记录排除原因（PRISMA流程图）
- 双人独立筛选（如资源允许）

**文献管理工具**
- Zotero：免费开源、插件丰富、浏览器一键采集
- Mendeley：Elsevier旗下、PDF管理强大、社交功能
- EndNote：商业软件、期刊样式丰富、与Word集成
- RefWorks：网页版、团队协作方便
- Citavi：德语区流行、任务管理功能

### 4. 高级检索技巧

**引文追溯法**
- 向前追溯：查看参考文献列表
- 向后追溯：利用Google Scholar的"被引用次数"功能
- 雪崩抽样：从一篇关键文献出发逐步扩展

**作者追踪**
- 识别领域内关键研究者
- 追踪其最新发表
- 使用ResearchGate、Google Scholar个人主页

**期刊追踪**
- 识别领域核心期刊
- 设置期刊目录推送（Table of Contents Alert）
- 关注特刊/专题

**预印本检索**
- arXiv（物理、数学、计算机）
- bioRxiv（生物学）
- medRxiv（医学）
- SSRN（社会科学）
- ChemRxiv（化学）
- 注意：预印本尚未经过同行评审

**灰色文献**
- 政府报告、政策文件
- 学位论文（ProQuest Dissertations）
- 会议摘要
- 组织白皮书
- 专利文献

### 5. 检索结果评估

**文献质量评估**
- 期刊影响因子和分区（JCR、中科院分区）
- 引用次数和h指数
- 同行评审状态
- 作者机构声誉
- 研究资金来源和利益冲突声明

**检索全面性评估**
- 多数据库交叉验证
- 不同检索策略结果对比
- 检索饱和判断（新增检索不再产生新文献）
- 记录检索日期和检索式（可复现性）

## 工作流程

### 标准检索流程

1. **需求分析**
   - 明确研究主题和问题
   - 确定时间范围和语言要求
   - 了解已有检索基础

2. **策略制定**
   - 分解研究问题为概念
   - 提取关键词和同义词
   - 选择目标数据库
   - 构建检索式

3. **执行检索**
   - 在各数据库中执行检索
   - 记录检索结果数量
   - 调整检索策略（如结果过多或过少）

4. **筛选管理**
   - 应用纳入排除标准
   - 使用文献管理工具整理
   - 生成最终文献清单

5. **结果报告**
   - 检索策略文档化
   - 文献清单（含元数据）
   - 检索统计信息
   - 后续建议

### 检索式模板

```
# 基本检索式结构
(关键词1 OR 同义词1 OR 同义词2) 
AND 
(关键词2 OR 同义词3) 
AND 
(关键词3 OR 同义词4)

# PubMed示例
("machine learning"[MeSH Terms] OR "deep learning" OR "neural network*" OR "artificial intelligence") 
AND 
("medical imaging"[MeSH Terms] OR "radiology" OR "diagnostic imaging") 
AND 
("review"[Publication Type] OR "systematic review"[Publication Type])

# Web of Science示例
TS = (("deep learning" OR "neural network*" OR "convolutional neural network*") 
AND 
("image segmentation" OR "semantic segmentation" OR "instance segmentation")) 
AND 
PY = (2020-2026)
```

## 使用指南

### 如何开始

1. **提供研究主题**：清晰描述你的研究领域或具体问题
2. **说明已有基础**：如果已经有部分文献或检索经验，请告知
3. **指定要求**：时间范围、语言、文献类型等特殊要求
4. **选择深度**：
   - 快速检索：1-2个数据库，核心关键词
   - 全面检索：多数据库，完整检索策略
   - 系统检索：完整PRISMA流程，可复现检索

### 注意事项

- 检索策略需要根据初步结果动态调整
- 不同数据库的语法和字段标签有所不同
- 预印本和灰色文献需谨慎评估质量
- 保留完整的检索记录以确保可复现性
- 定期更新检索以跟踪最新研究进展

### 最佳实践

- 使用受控词汇表（如MeSH）提高查全率
- 同时使用关键词和主题词提高查准率
- 先进行小规模测试检索，再正式执行
- 记录每一次检索调整和结果变化
- 考虑合作者或图书馆员的检索专业意见
- 定期备份文献管理数据
